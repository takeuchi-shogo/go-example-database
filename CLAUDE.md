# Claude.md

## 1. プロジェクトの説明

このプロジェクトは、Golangで実装されたデータベースです。

### フェーズの説明

現在のフェーズは、データベースの基本機能の実装を行っています。

**Goal**:

- TiDB や CockroachDB などのオープンソースのデータベースを参考に、分散型の本格的なデータベースを実装できるようになること
  - [TiDB](https://github.com/pingcap/tidb)
  - [CockroachDB](https://github.com/cockroachdb/cockroach)
- Golang の基本的な文法を理解し、応用できるように実際にコードを書いてみること

## プロジェクトの技術スタック

- Golang

## コーディング規則

- コードは Golang のコーディング規約に従う
- 関数は小さく、シンプルで、読みやすいものにする
- ドキュメントコメントは簡潔に、でも詳細にわかりやすいように書く
- インラインコメントは適宜使用する
- 命名については、わかりやすく、一貫性のあるものにする。c や f などの短縮語は使用しない
- プライベート、パブリックな関数や変数、フィールドの判断を迷う場合は基本的にプライベートで実装する。明らかにパブリックなものはパブリックで実装して良い。

## PR について

プルリクエストとコミット記録を生成する際は

以下の形式に従ってください：

- 件名と本文を空白行で区切ってください。
- コミット記録の本文で、変更前の状態、変更内容、変更理由を説明してください。
- 件名には、変更の大部分が発生したパッケージ名を接頭辞として付加してください。
- 複数コミットを含むプルリクエストの場合、各コミットの内容をPRレコード内で要約してください。
- ユーザーから明示的に求められない限り、テスト計画を含めないでください。

## 対話スタイル

- 率直かつ誠実であること。
- 不要な挨拶は省略すること。
- 誤りがある場合は指摘し、理由を説明すること。
- 改善の余地がある場合はより良い代替案を提案すること。
- 正確性と効率性に重点を置くこと。
- 必要に応じて私の前提に疑問を呈すること。
- 質の高い情報と率直さを優先すること。
