# Phase 1: ストレージエンジン 実装レポート

## 概要

Phase 1 ではデータベースの基盤となるストレージエンジンを実装しました。

## 実装ファイル (`internal/storage/`)

| ファイル | 役割 |
|----------|------|
| `page.go` | ページ構造体 |
| `pager.go` | ファイル ↔ ページの読み書き |
| `page_layout.go` | スロット配列方式のページレイアウト |
| `types.go` | カラム型定義（Int32, Int64, String, Bool） |
| `schema.go` | スキーマ定義 |
| `row.go` | 行のシリアライズ/デシリアライズ |
| `table.go` | テーブル操作（Insert, Scan） |

## 主要概念

### ページ (Page)

- 固定サイズのデータブロック（通常 4KB）
- ディスクとメモリ間のデータ転送の最小単位

### Pager

- ページの読み書きを管理
- ファイルI/Oの抽象化

### スキーマ (Schema)

- テーブルの構造定義
- カラム名と型の組み合わせ

### 行 (Row)

- 1レコードのデータ
- シリアライズ: メモリ → バイト列
- デシリアライズ: バイト列 → メモリ

## ゴール

テーブルへの Insert と Scan（全件取得）が動作すること
